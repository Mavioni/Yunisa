# /7p — Prior Perfect Planning Workflow

// turbo-all

## Phase 1 — Freshen the GitNexus Index

Ensure the call graph is current before querying it.

```powershell
cd c:\Users\massi\yunisa
npx gitnexus analyze
```

## Phase 2 — Query Relevant Execution Flows

Ask GitNexus which symbols and processes are related to the task.  
Replace `<TASK_CONCEPT>` with the user's task in plain English.

```powershell
npx gitnexus query "<TASK_CONCEPT>"
```

## Phase 3 — Blast Radius Pre-flight

For each symbol you plan to change, run impact analysis.  
Replace `<SYMBOL>` with the actual function/class name.

```powershell
npx gitnexus impact --symbol "<SYMBOL>" --direction upstream
```

> [!IMPORTANT]
> If impact returns HIGH or CRITICAL risk, STOP and warn the user before proceeding.

## Phase 4 — Server Health Check

Confirm the local inference pipeline is healthy.

```powershell
try {
  Invoke-WebRequest -Uri http://127.0.0.1:8080/health -TimeoutSec 3 |
    Select-Object -ExpandProperty Content
} catch { "Server not reachable — proceeding with planning only." }
```

## Phase 5 — Ask YUNISA to Critique the Plan

If the server is online, send the draft plan to YUNISA's own LLM for a critique.
Replace `<DRAFT_PLAN>` with a one-paragraph summary of your intended approach.

```powershell
$body = @{
  messages = @(@{
    role = "user"
    content = "Critique this engineering plan and suggest improvements: <DRAFT_PLAN>"
  })
} | ConvertTo-Json -Depth 5

Invoke-RestMethod -Uri http://127.0.0.1:8080/v1/chat/completions `
  -Method Post -ContentType "application/json" -Body $body |
  Select-Object -ExpandProperty choices | ForEach-Object { $_.message.content }
```

## Phase 6 — MSAM Memory Recall

Check if YUNISA has long-term memory entries related to this task.

```powershell
$body = @{ query = "<TASK_CONCEPT>"; limit = 5 } | ConvertTo-Json
Invoke-RestMethod -Uri http://127.0.0.1:8080/msam/recall `
  -Method Post -ContentType "application/json" -Body $body
```

If the MSAM endpoint is unavailable (server offline), skip this step.

## Phase 7 — Formalize the Plan

Based on all research above:

1. Write an `implementation_plan.md` artifact summarising:
   - What will change, and why
   - Blast radius (affected symbols, d=1 / d=2 / d=3)
   - LLM critique summary and adjustments made
   - Step-by-step execution checklist

2. Request user review via `notify_user` and **wait for LGTM** before writing any code.
